# Claims sharing — supporting evidence

Background for the **`ODEDentalClaim`** profile (the claims-ready, non-financial
ExplanationOfBenefit). The profile itself is defined in the four interface views; these are
the documents that justify its shape.

| Document | What it proves |
|----------|----------------|
| [`design-rationale.md`](./design-rationale.md) | Why the profile is non-financial, why it inherits CARIN BB, and what it deliberately declines to decide |
| [`CMS1500-crosswalk.md`](./CMS1500-crosswalk.md) | Box-by-box mapping against the CMS-1500 form — 9 direct, 8 indirect, 5 deliberately excluded, and the **named remaining gaps** |
| [`evidence-sufficiency.md`](./evidence-sufficiency.md) | Field-by-field proof that every requirement **CMS and Humana actually state, in their own words**, is carried by a named element — so a compliant submission can be generated via either pathway |
